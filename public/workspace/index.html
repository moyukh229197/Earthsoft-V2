Guide & Logic

How Earthsoft works, which inputs drive the model, how quantities are produced, and how the workflow moves from setup to export.

1. What Earthsoft Does

Earthsoft is a corridor engineering workspace for railway earthwork planning. It combines chainage-based levels, structural inputs, station-yard geometry, map alignment, and estimating tools so one project can move from raw data to a report-ready output without shifting between separate spreadsheets and sketches.

  • Imports level data and applies bridge, curve, and yard effects to the corridor model.
  • Computes chainage-wise fill, cut, blanket, prepared subgrade, and earthwork quantities.
  • Supports map review, quantity diagnostics, land acquisition review, and estimate generation.

2. Recommended Workflow

The intended flow is Setup -> Inputs -> Analysis -> Estimates -> Export. A reliable sequence is:

  1. Create or open the project and import the level file first so chainage computation has a base dataset.
  2. Complete design inputs by loading bridges, curves, stations and yards, then KML/KMZ if map review is required.
  3. Run Verify to generate the chainage-wise earthwork model and unlock review tools.
  4. Review the outputs in Earthwork Table, Graphs, Roll Diagram, Alignment Map, and Diagnostic Minimap.
  5. Prepare delivery quantities in Estimates, Earthwork Structures, Land Acquisition, and P-Way Calculation.
  6. Export the report once the project is verified and the planning quantities have been reviewed.

3. Core Calculation Logic

Earthsoft calculates earthwork section by section along the alignment. The level file supplies chainage, existing ground, and proposed formation. Bridge, curve, and station-yard inputs then modify the effective corridor width or deduct structure ranges where needed. The engine inserts intermediate chainages around structural boundaries so quantities are segmented cleanly instead of being averaged across incompatible zones.

Volume = (Area1 + Area2) / 2 * Distance
  • Fill and cut are derived from the relationship between formation level and ground level at each chainage.
  • Curves can trigger widening logic so the formation width is not treated as constant across the whole route.
  • Bridges and structure ranges can segment the corridor and influence deduction logic.
  • Stations and yards extend track spacing, loop limits, and platform effects into the earthwork model.

4. Earthwork Structures & Estimates Logic

The Earthwork Structures page feeds the auto-estimation for 1130 - Earthwork & Drains. Quantities are calculated from chainage ranges, side, and type, then pushed directly into the estimate workbook structure.

  • Retaining Walls: Quantity is calculated as running metre from To Chainage - From Chainage, multiplied by side. Both counts both sides.
  • Wall Types: The page currently supports 3.0 m and 4.0 m high retaining walls, matching the 1130 workbook items.
  • Auto Suggest Retaining Walls: Suggestions are generated only after KML alignment import and earthwork calculation. The app scans wall-worthy earthwork zones and checks nearby mapped constraints such as roads, waterbodies / drainage edges, and habitations / buildings.
  • Retaining Wall Heuristic: A wall is suggested when the alignment side appears constrained and the nearby mapped feature would likely make normal side slopes or additional land acquisition undesirable. These suggestions are marked Review Required and are planning aids, not final design decisions.
  • Drains: Quantity is calculated from chainage length, side, and number of runs. The summary table bifurcates drain quantity by type and side.
  • Auto Suggest Drains: The app derives drain candidates from earthwork geometry and yard extents:
    Side drain: for cutting depth greater than 4 m
    Berm drain: where berm rules trigger in cutting
    Catchwater drain: for embankment height greater than 6 m
    Yard drain: across station loop / yard extents
  • 1130 Linkage: Retaining wall totals are mapped into 1131-B Walling and drain totals into 1131-C Side Drains. Earthwork quantities continue to come from calculated cross-sections.
  • 1110 Linkage: Preliminary Survey Expenses are auto-calculated from total main-line route length using the selected survey type.
  • 1120 Linkage: Land Acquisition uses corridor width rules, zone rates, solatium, and multiplier settings to generate planning-level land cost.
  • P-Way Linkage: Permanent way quantities can be prepared from station-yard geometry and track layout data on the P-Way page.

5. Review Tools & Diagnostics

Once calculation is complete, Earthsoft exposes several review surfaces so design decisions are not made from one table alone. The Overview page now acts as the workflow home, and the Diagnostic Minimap sits inside the review stage as a quick corridor-health scanner.

  • Earthwork Table: Review chainage-wise computed values, quantities, and anomalies in one grid.
  • Graphs and Roll Diagram: Cross-check profile behaviour, cut-fill distribution, and the route story visually.
  • Alignment Map: Review the imported KML/KMZ route with earthwork overlays, land boundary, stations, and map-based planning hints.
  • Diagnostic Minimap: The barcode scanner reviews the corridor for severe fills, severe cuts, and selected spacing conflicts. Red segments are clickable and jump directly to the corresponding Earthwork Table region.
  • Verification: Export should be treated as the final step only after the model has been verified and reviewed across these outputs.

Workflow Reference

Setup & Inputs
Start with levels, then complete bridges, curves, and stations/yards. Add KML/KMZ when you need map-based review, land boundary visualization, and retaining-wall suggestions.
Core Formula
Volume = (Area1 + Area2) / 2 x Distance
This trapezoidal step is applied sequentially between adjacent cross-sections, with structure boundaries inserted where needed.
Auto-Suggestion Rules
• Side drain: cutting > 4 m
• Catchwater drain: embankment > 6 m
• Berm drain: berm-triggered cutting zones
• Retaining walls: review-based suggestions where mapped constraints indicate limited practical widening
Delivery Rule
Use Verify before export. The safest workflow is to check Overview, Earthwork Table, Graphs, Roll Diagram, Alignment Map, and estimate inputs before producing the final report.
