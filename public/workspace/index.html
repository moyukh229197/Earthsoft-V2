Earthsoft Usage Guide

1. Project Overview

Earthsoft provides an integrated solution for Railway Earthwork Calculations. It handles cross-sectional data, formation levels, bridges, curves, and loops to accurately compute earthwork volumes and generate BOQ estimates based on USSR rates.

2. Data Import Sequence

To build a complete model, import CSV/Excel files in this optimal order:

  1. Project Data (.ew): Load an existing project workspace.
  2. Levels Data: Contains chainage, ground levels, and proposed levels. This generates the core earthwork quantities.
  3. Bridges: Defines bridge start/end chainages. Areas intersecting bridges will be automatically handled as deductions in the calculation engine.
  4. Curves: Defines railway curve radii. This applies extra formation width on curves.
  5. Loops & Platforms: Applies additional track centers and platform widths for station yards.

3. Calculation Methodology

Volumes are calculated using the Trapezoidal Rule applied sequentially between adjacent cross-sections. When structures (Bridges, Stations, Curves) overlap, the calculation engine dynamically inserts sub-chainages to segment the earthwork calculation perfectly along structure boundaries.

Volume = (Area1 + Area2) / 2 * Distance

4. Verifying & Exporting

Use the Verify button in the top toolbar to validate the calculation engine outputs. Once validated, you can view the 'Calculation Table', visualizations, and 'Estimates' tabs. Use Export Report to generate a comprehensive PDF calculation sheet.

5. Diagnostic Minimap (Clash Detection)

The integrated Diagnostic Minimap appears at the bottom of the Overview screen after calculation. It acts as an automated tracking barcode, giving you a bird's-eye view of your corridor's engineering health.

  • Automated Scanning: The timeline barcode is broken down into 100 equal segments. The calculator scans every chainage mathematically to identify extreme earthwork heights (Cut > 8m or Bank > 8m) or foul track clearance spacing (< 4.72m from main line).
  • Visual Alerts: Safe sections remain muted, while problematic segments are painted instantly in Red.
  • Interactive Navigation: Click on any red error bar to immediately jump to the Calculation Table and auto-scroll to the exact chainage causing the problem, highlighting the specific row for manual investigation.

6. Visual P-Way Editor (BETA)

The Visual Editor allows you to parametrically design track layouts (Loops, Platforms, Sidings, etc.) visually with node-based logic.

  • Nodes & Canvas: Drag components (Tracks, Turnouts, Platforms) from the left sidebar onto the infinite grid canvas. Pan the canvas by clicking and dragging on the background.
  • Configuring Components: Each component asks for specific inputs. For example, a Loop / Siding node needs a Takeoff Chainage and Length, and a Platform needs a built Width and Location.
  • Wiring: To map logic, click and drag from the green/blue output dot (right side of a node) to the input dot (left side) of an adjacent node. This wires the topological flow.
  • Compiling layout: Once built, hit the Compile Layout button. The engine parses the visual nodes and auto-injects the generated chainage boundaries directly into your project's Loops & Stations tab, auto-calculating earthwork across your new yard.
